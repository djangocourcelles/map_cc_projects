Type ProjectRecord (objet retourné par scannerWorkspace()) :
```javascript
{
  id: string,            // nom du dossier (clé unique)
  nom: string,           // nom du dossier (affiché)
  chemin: string,        // chemin absolu du projet
  has_git: boolean,      // .git/ existe
  stacks: Array<{        // tableau vide si fallback dossier
    stack: string,       // ex. 'Node.js', 'Python'
    emoji: string|null   // ex. '🟢', '🐍', null si fallback
  }>,
  branche: string|null,  // 'main', 'dev', null si sans git
  jours_depuis_commit: number|null,
  date_relative: string, // ex. "il y a 3 jours", "sans git"
  rayon: number,         // 28 à 56 (formule D-10)
  opacite: number,       // 0.3 | 0.6 | 1.0 | 0.4 (sans git)
  has_gsd: boolean,      // .planning/ existe
}
```


<threat_model>
## Frontières de confiance

| Frontière | Description |
|-----------|-------------|
| filesystem → scanner | Noms de dossiers lus depuis un FS local — données non filtrées passées à `execSync` |
| API → browser | JSON exposé sur localhost sans auth |

## Registre STRIDE

| Threat ID | Catégorie | Composant | Disposition | Plan de mitigation |
|-----------|-----------|-----------|-------------|-------------------|
| T-01-01 | Injection | `scanner.js` / `execSync` | mitigate | Le `cwd` passé à `execSync` est construit avec `path.join(WORKSPACE, e.name)` depuis `fs.readdirSync`. Le nom de dossier ne peut pas contenir de séquences shell car `execSync` reçoit une string de commande et non un tableau. Risque résiduel minimal : workspace local, pas de données injectées par un tiers. |
| T-01-02 | Information Disclosure | `GET /api/projects` | accept | Expose chemins absolus locaux. Usage 100% local (localhost), pas de réseau externe. Risque acceptable. |
| T-01-03 | Denial of Service | `server.js` / scanner synchrone | accept | Le scan de 26 dossiers est synchrone au démarrage. Durée estimée < 2s. Acceptable pour un outil local à usage personnel. |
</threat_model>
